Defense in Depth

About This Model

Defense in depth is the practice of layering multiple security controls so that if one fails, others remain. Think medieval castle: moat, walls, gates, guards, keep. Each layer slows the attacker and increases the chance of detection.

Why it matters: No single security control is perfect. Firewalls can be bypassed, authentication can be compromised, encryption can be broken. Layered defenses ensure that a single failure doesn't mean total compromise.

Key Insights

  • Assume breach: Defense in depth starts with the assumption that attackers will get through the first layer. The question is: what happens next?
  • Layers slow attackers: Each layer adds time, complexity, and risk of detection. Attackers need to bypass 5 layers instead of 1â€”that's 5 chances to catch them.
  • Diversity matters: Don't use the same vendor/technology for all layers. If one has a vulnerability, you don't want all layers to fall at once.
  • Monitoring is a layer: Detection and response are as important as prevention. You can't prevent every attack, but you can detect and contain them.
  • Real-world example: Sony PlayStation Network breach (2011) had weak perimeter security, no encryption at rest, and poor monitoring. All layers failed. 77 million accounts compromised. Cost: $171M.
